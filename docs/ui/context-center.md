## 3. 文档上传流程

```
点击 [上传文档] → 拖拽/选择文件
    ↓
显示上传进度条
    ↓
上传完成 → 显示状态: PROCESSING
    ↓
后台: Tika 提取 → 分块 → Embedding → Milvus 写入
    ↓
状态变为: INDEXED（自动刷新）
```

## 5. 需要人工评审的设计决策

| 决策点 | 选项 A | 选项 B | 建议 |
|--------|--------|--------|------|
| **检索交互** | 即时检索（输入即搜） | 回车触发 | **待评审**: A 体验好但请求多，B 省流量 |
| **结果展示** | 列表 | 卡片网格 | **待评审**: 列表适合长摘要 |
| **原文查看** | 新页面 | 侧滑抽屉 | **待评审**: 侧滑更流畅 |
| **文件夹层级** | 树形结构 | 扁平 + 标签 | **待评审**: 树形更符合文件管理习惯 |

## 6. API 依赖

```
POST   /api/context/documents/upload    → 上传文档
GET    /api/context/documents           → 文档列表
GET    /api/context/documents/{id}      → 文档详情
DELETE /api/context/documents/{id}      → 删除文档
POST   /api/context/search              → RAG 检索
GET    /api/context/workspaces          → 工作空间列表
```
