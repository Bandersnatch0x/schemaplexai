## 5. 需要人工评审的设计决策

| 决策点 | 选项 A | 选项 B | 建议 |
|--------|--------|--------|------|
| **成本展示精度** | 2 位小数 | 4 位小数 | **待评审**: 2 位适合展示，4 位适合财务对账 |
| **监控刷新频率** | 实时 WebSocket | 每 30 秒轮询 | **待评审**: WebSocket 实时性好但连接多 |
| **告警通知渠道** | 站内信 + 邮件 | + 企业微信/Slack | **待评审**: 企业环境优先企业微信 |

## 6. API 依赖

```
GET    /api/ops/costs/summary           → 成本汇总
GET    /api/ops/costs/trend             → 成本趋势
GET    /api/ops/costs/detail            → 成本明细
POST   /api/ops/budgets                 → 创建预算
GET    /api/ops/monitor/services        → 服务监控
GET    /api/ops/monitor/infrastructure  → 基础设施监控
GET    /api/ops/notifications           → 通知列表
POST   /api/ops/notifications/read      → 标记已读
```
