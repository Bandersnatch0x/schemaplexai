### 编辑器功能

| 功能 | 说明 |
|------|------|
| Markdown 编辑 | 支持标准 Markdown + GFM |

## 6. 需要人工评审的设计决策

| 决策点 | 选项 A | 选项 B | 建议 |
|--------|--------|--------|------|
| **编辑器选型** | 自研 Markdown 编辑器 | 集成 Vditor / Milkdown | **待评审**: B 功能丰富但包体积大 |
| **实时协作** | 单人编辑 + 锁机制 | 多人实时协作（OT/CRDT） | **待评审**: MVP 阶段建议 A |
| **版本对比样式** | 左右双栏 | 内联单栏 | **待评审**: 内联适合小改动，双栏适合大改动 |
| **Steering 绑定 Agent** | 一对一 | 一对多 | **待评审**: 一对多更灵活 |

## 7. API 依赖

```
GET    /api/specs                   → Spec 列表
POST   /api/specs                   → 创建 Spec
GET    /api/specs/{id}              → Spec 详情
PUT    /api/specs/{id}              → 更新 Spec
POST   /api/specs/{id}/publish      → 发布版本
GET    /api/specs/{id}/versions     → 版本列表
GET    /api/specs/versions/diff     → 版本对比
POST   /api/specs/{id}/reviews      → 提交评审
GET    /api/specs/{id}/reviews      → 评审列表
```
