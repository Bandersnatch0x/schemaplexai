## 5. 交互流程

### 5.1 创建节点

```
从左侧面板拖拽节点 → 放置到画布
    ↓
自动弹出属性配置面板
    ↓
配置完成 → 节点显示在画布
```

### 5.2 连接节点

```
点击源节点输出点 → 拖拽到目标节点输入点
    ↓
显示连接箭头（Bezier 曲线）
    ↓
条件分支显示条件标签
```

### 5.3 测试运行

```
点击 [测试] → 弹出参数输入框
    ↓
输入测试参数 → 触发执行
    ↓
画布节点实时显示状态（运行中/成功/失败）
    ↓
底部面板显示执行日志
```

## 7. 需要人工评审的设计决策

| 决策点 | 选项 A | 选项 B | 建议 |
|--------|--------|--------|------|
| **画布引擎** | React Flow（轻量） | X6 / LogicFlow（功能丰富） | **待评审**: React Flow 社区活跃，X6 阿里生态功能强 |
| **布局算法** | 手动拖拽 | 自动布局（Dagre） | **待评审**: 建议两者结合 |
| **节点样式** | 图标 + 文字 | 缩略图预览 | **待评审**: MVP 建议 A |
| **Undo/Redo** | 完整历史栈 | 仅自动保存 | **待评审**: 建议完整历史栈 |

## 8. API 依赖

```
GET    /api/workflow/templates          → 模板列表
POST   /api/workflow/templates          → 创建模板
PUT    /api/workflow/templates/{id}     → 更新模板
POST   /api/workflow/templates/{id}/publish → 发布
POST   /api/workflow/instances          → 创建并执行实例
GET    /api/workflow/instances/{id}     → 执行详情
POST   /api/workflow/instances/{id}/cancel → 取消
```
