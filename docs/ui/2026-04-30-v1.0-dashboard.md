## 1. 页面定位

Dashboard 是用户登录后的默认页面，提供平台运行状态的一目了然概览。

**目标用户**: 所有角色（管理员、开发者、业务用户）
**核心功能**: 快速了解 Agent 运行状态、Token 消耗、待办事项

## 3. 组件拆分

| 组件 | 职责 | 复用性 |
|------|------|--------|
| **StatCard** | 统计卡片（数值 + 趋势箭头） | 高（通用） |
| **TrendChart** | 折线图（Ant Design Charts） | 高（通用） |
| **PieChart** | 饼图/环形图 | 高（通用） |
| **ExecutionTable** | 执行记录表格 | 中（AgentExecutor 复用） |
| **AgentRanking** | Agent 排行列表 | 低（Dashboard 专用） |

## 4. 交互流程

### 4.1 页面加载

```
用户登录 → 自动跳转 /dashboard
    ↓
并行请求:
  ├─ GET /api/agents/stats          → 统计卡片
  ├─ GET /api/executions/trend      → 趋势图
  ├─ GET /api/costs/distribution    → 饼图
  ├─ GET /api/executions/recent     → 最近记录
  └─ GET /api/agents/top            → Agent 排行
    ↓
Skeleton 占位 → 数据填充 → 图表渲染
```

### 4.2 实时更新

- 统计卡片: 每 60 秒自动刷新
- 最近记录: 通过 SSE 监听执行完成事件，自动追加到列表顶部
- 图表: 手动刷新按钮（避免频繁重绘）

### 4.3 点击交互

| 元素 | 点击行为 |
|------|----------|
| 统计卡片 | 跳转对应详情页 |
| 执行记录行 | 打开执行详情 Drawer |
| Agent 排行项 | 跳转 Agent 执行器 |
| [查看全部] | 跳转对应列表页 |

## 5. 响应式适配

| 断点 | 布局变化 |
|------|----------|
| ≥ 1440px | 4 列统计卡片 + 双列图表 + 表格 |
| ≥ 1024px | 2 列统计卡片 + 单列图表堆叠 + 表格 |
| < 1024px | 单列堆叠 + 侧边栏收起 |

## 6. 需要人工评审的设计决策

| 决策点 | 选项 A | 选项 B | 建议 |
|--------|--------|--------|------|
| **实时更新策略** | 全页自动刷新（60s） | 仅 SSE 推送 + 手动刷新 | **待评审**: A 简单但性能差，B 体验好但实现复杂 |
| **图表库选型** | Ant Design Charts（已用） | ECharts / D3.js | **待评审**: 当前已用 Ant Design Charts，是否满足复杂图表？ |
| **数据保留周期** | 最近 7 天 | 最近 30 天 | **待评审**: 影响 API 查询性能 |

## 7. API 依赖

```
GET /api/agents/stats              → { totalAgents, todayExecutions, tokenUsage, pendingApprovals }
GET /api/executions/trend?days=7   → [{ date, count, successCount }]
GET /api/costs/distribution        → [{ model, cost, percentage }]
GET /api/executions/recent?limit=5 → ExecutionRecord[]
GET /api/agents/top?limit=5        → [{ agentId, name, executionCount, successRate }]
```
