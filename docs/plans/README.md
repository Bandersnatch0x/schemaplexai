# Plans / 实施计划

实施计划是设计到代码的桥梁。每个计划必须明确任务分解、责任人、验收标准和工期。

## 范围

- 迭代计划（Sprint Plan）
- 技术预研计划
- 迁移/重构计划
- Plugin 生成的 implementation plan（经评审后归档于此）

## 格式

```
YYYY-MM-DD-<topic>-plan.md
```
