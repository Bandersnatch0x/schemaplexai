# Designs / 架构设计文档

架构设计是编码的前置条件。新模块或重大重构必须有设计评审通过后才能进入实现阶段。

## 范围

- 系统架构设计
- 模块级详细设计
- 数据流与时序图
- 部署架构

## 命名规范

```
<topic>.md
```

`topic` 为短横线分隔的英文小写主题词，与对应 `specs/`、`plans/`、`ui/` 文档保持一致。
