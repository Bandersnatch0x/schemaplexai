# MCP Tool Discovery Service Spec

## Overview

Periodic background service that discovers tools from configured MCP servers and registers them in the global `ToolRegistry`.

## Requirements

- Query approved MCP servers (`status = 1`) from the database
- Discover tools from each server in parallel using a fixed thread pool (size 4)
- Isolate failures per-server so one unreachable server does not block others
- Apply guardrails validation to each tool description before registration
- Respect server tool whitelist (null/empty means all tools allowed)
- Skip already-registered tools
- Run periodically via Spring `@Scheduled` with configurable interval (default 60s)

## Design

### Qualified Name Format

```
mcp:<serverId>:<toolName>
```

Example: `mcp:1:read_file`

### Parallel Discovery

- `CompletableFuture.runAsync(...)` with a daemon thread pool
- `.exceptionally(...)` per-server error isolation
- `CompletableFuture.allOf(...).join()` to wait for all servers

### Guardrails

- `GuardrailsEngine.validateInput(tool.description())` must return `success = true`
- Failed tools are logged and skipped

## API

### `syncAll()`

Scheduled entry point. Fetches approved servers, discovers tools in parallel, and registers them.

### `discoverForServer(SfMcpServer server)`

Returns `List<ToolDefinition>` for a single server without registering. Exposed for testing.

## Dependencies

- `McpServerMapper` — query approved servers
- `McpClientManager` — create/get cached `McpClient`
- `ToolRegistry` — register discovered tools
- `GuardrailsEngine` — validate tool descriptions

## Testing

- Unit tests with JUnit 5 + Mockito
- Mock all external dependencies
- Test: happy path, guardrails failure, whitelist filtering, disconnected client, per-server failure isolation, no servers, existing tool skip
